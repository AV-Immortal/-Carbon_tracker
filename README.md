# -Carbon_tracker
# 低碳行为追踪系统

一个简单易用的个人碳排放追踪工具，帮助你记录和分析日常生活中的碳排放量。

## 功能特性

- ✅ **数据录入**：记录每日用电量、用水量和通勤方式
- ✅ **碳排放计算**：自动计算单日和累计碳排放量
- ✅ **数据持久化**：自动保存到本地CSV文件，易于扩展
- ✅ **历史查询**：查看所有历史记录和指定日期范围的数据
- ✅ **趋势分析**：查看碳排放趋势，了解变化情况
- ✅ **统计分析**：通勤方式使用统计、平均排放量等

## 系统要求

- Python 3.6 或更高版本
- 无需额外依赖（仅使用Python标准库）

## 安装使用

### 1. 下载程序

将 `carbon_tracker.py` 文件下载到任意目录。

### 2. 运行程序

在命令行中执行：

```bash
python carbon_tracker.py
```

### 3. 使用菜单

程序启动后会显示主菜单，根据提示选择相应功能：

```
==================================================
           低碳行为追踪系统
==================================================
1. 添加每日数据
2. 查看单日碳排放
3. 查看累计碳排放
4. 查看所有历史数据
5. 查看碳排放趋势（最近7天）
6. 查看通勤方式统计
7. 删除记录
0. 退出
==================================================
```

## 使用说明

### 添加每日数据

选择菜单选项 `1`，按提示输入：

- **日期**：格式为 YYYY-MM-DD（如 2024-01-15），留空则使用今天
- **用电量**：单位为 kWh（千瓦时）
- **用水量**：单位为 m³（立方米）
- **通勤方式**：从列表中选择（步行、自行车、地铁、公交、私家车、电动车等）
- **通勤距离**：单位为 km（公里）

系统会自动计算当日的碳排放量并保存。

### 查看单日碳排放

选择菜单选项 `2`，输入日期即可查看该日的详细数据和碳排放量。

### 查看累计碳排放

选择菜单选项 `3`，可以：
- 查看所有记录的累计碳排放
- 指定日期范围查看该时间段内的累计排放

### 查看历史数据

选择菜单选项 `4`，查看所有历史记录的详细列表。

### 查看碳排放趋势

选择菜单选项 `5`，查看最近N天的：
- 碳排放趋势（上升/下降）
- 最高/最低排放量
- 平均排放量

### 查看通勤方式统计

选择菜单选项 `6`，查看各种通勤方式的使用次数和占比。

### 删除记录

选择菜单选项 `7`，输入日期即可删除该日的记录。

## 数据存储

所有数据自动保存在 `carbon_data.csv` 文件中，文件格式如下：

```csv

```

你可以：
- 直接用Excel或其他工具打开查看
- 备份文件以防数据丢失
- 导入到其他系统进行进一步分析

## 碳排放计算方法

### 计算公式

```
总碳排放 = 用电碳排放 + 用水碳排放 + 通勤碳排放

用电碳排放 = 用电量(kWh) × 0.581
用水碳排放 = 用水量(m³) × 0.001
通勤碳排放 = 通勤距离(km) × 通勤方式系数
```

### 通勤方式碳排放系数

| 通勤方式 | 系数 (kg CO2/km) |
|---------|-----------------|
| 步行     | 0.0             |
| 自行车   | 0.0             |
| 地铁     | 0.04            |
| 公交     | 0.089           |
| 私家车   | 0.192           |
| 电动车   | 0.053           |
| 出租车   | 0.192           |
| 网约车   | 0.192           |

*注：以上系数为参考值，可根据实际情况调整*

## 常见问题

### Q: 如何修改碳排放系数？

A: 修改 `CarbonTracker.EMISSION_FACTORS` 字典中的值即可。

### Q: 数据文件在哪里？

A: 默认保存在程序同目录下的 `carbon_data.csv` 文件中。

### Q: 如何备份数据？

A: 直接复制 `carbon_data.csv` 文件即可。

### Q: 可以在手机上使用吗？

A: 当前版本为命令行程序，需要在电脑上运行。未来可扩展为移动应用。

### Q: 支持多用户吗？

A: 当前版本为单用户模式。可通过添加用户字段扩展为多用户支持。
